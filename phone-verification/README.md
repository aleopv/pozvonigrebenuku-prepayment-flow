# Подтверждение номера

Автономная страница между экраном стоимости подписки в Telegram-боте и страницей документов и согласий.

Основной Telegram-бот: `@grebenyuk_call_bot`. Бот поддержки: `@grebenyuk_help_bot`.

Для проверки и форматирования международных номеров локально подключена `libphonenumber-js` с полной метаинформацией по странам. CDN не используется.

## Пользовательский flow

```text
Telegram: стоимость подписки
→ ввод номера
→ проверка номера
→ ввод кода
→ номер подтверждён
→ документы и согласия
→ Telegram: готовность к оплате
```

Ошибки формата номера, уже занятого номера и неверного кода показываются попапом поверх текущего шага. Они не создают дополнительные этапы flow.

На первом шаге кнопка «Назад» возвращает в бот на состояние `subscription_price`. После успешного подтверждения «Продолжить» открывает `consent-landing`.

## Локальный запуск

Из корня общего проекта:

```powershell
npx --yes http-server -p 4173 -a 127.0.0.1 -c-1 .
```

Открыть:

```text
http://127.0.0.1:4173/phone-verification/
```

В демонстрационном режиме код подтверждения:

```text
1234
```

Для тестового испанского номера `+34624494031` используй код:

```text
4031
```

Любой другой код покажет попап «Неверный код».

## Настройка

Параметры находятся в `config.js`:

- `requestCodeUrl` — endpoint отправки SMS или запуска звонка;
- `checkPhoneUrl` — endpoint проверки, можно ли привязать номер;
- `verifyCodeUrl` — endpoint проверки кода;
- `consentPageUrl` — адрес страницы согласий;
- `defaultBackUrl` — возврат в Telegram на экран стоимости;
- `supportUrl` — служба заботы;
- `demoCode` — код только для локальной демонстрации;
- `demoCodesByPhone` — временные тестовые коды для конкретных номеров;
- `demoOccupiedPhone` — номер, показывающий попап «Номер уже используется» без backend;
- `resendDelaySeconds` — пауза перед повторной отправкой.

Поддерживаемые query-параметры:

- `session_id`;
- `user_id`;
- `back_to` — возврат с первого шага;
- `consent_to` — адрес страницы согласий;
- `consent_back_to` — куда ведёт «Назад» со страницы согласий;
- `return_to` — куда страница согласий вернёт пользователя после принятия условий.

## Backend-контракт

Проверка доступности номера:

```json
{
  "phone": "+79991234567",
  "sessionId": "abc123",
  "userId": "42"
}
```

Ответ:

```json
{
  "available": true
}
```

Для уже используемого номера endpoint возвращает `{"available": false}` или HTTP `409`.

Отправка кода:

```json
{
  "phone": "+79991234567",
  "sessionId": "abc123",
  "userId": "42"
}
```

Ответ:

```json
{
  "verificationId": "verification-123"
}
```

Проверка кода:

```json
{
  "phone": "+79991234567",
  "code": "4031",
  "verificationId": "verification-123",
  "sessionId": "abc123",
  "userId": "42"
}
```

Успешный ответ:

```json
{
  "verified": true
}
```

В production переход к согласиям должен разрешаться только после серверной фиксации подтверждённого номера. Клиентский `localStorage` используется исключительно для демонстрации интерфейса.

При полном обновлении страницы flow намеренно сбрасывается: снова открывается первый шаг, а поле номера содержит только знак `+`.
